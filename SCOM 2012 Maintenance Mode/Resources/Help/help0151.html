Control Codes

Control codes allow you to specify maintenance options with the import file itself.
This allows quick changing of settings for different groups of servers.

For example, you may have a group of servers that routinely need a 5 hour maintenance window, and another group that only need 2 hours. You can set these time windows via a control code called #window.

Import File Control Codes

#window = xx

Where xx is the length in minutes. (min:10, max:86400)

#comment = xxxx

Replace xxxx with the text you want to use.

#whatif = TRUE/FALSE

Use either TRUE or FALSE

#auto-start

This control code does not have any options. If it is present in the import file, the application will automatically choose the "Start/Update" maintenance option as soon as the full list has been imported. It will then close.
This control code can be used for scheduling maintenance mode for future planned events where you may not be available to run this tool yourself.

Example Files

Below is an couple of examples of import files, showing the use of the control codes.

Example 1

Set the maintenance window to 2 hours, and check for cluster resources...

    #window = 120
    #cluster = TRUE

    server1.domain.local
    server2.domain.local
    server3.domain.local
    server4.domain.local

Example 2

Change the comment to reflect the reason for the maintenance time...

    #comment = Scheduled Windows Patching

    server1.domain.local
    server2.domain.local
    server3.domain.local
    server4.domain.local

Example 3

All control codes shown...

    #window = 120
    #comment = Scheduled Windows Patching
    #cluster = TRUE
    #whatif = FALSE
    #auto-start

    server1.domain.local
    server2.domain.local
    server3.domain.local
    server4.domain.local

Scheduled Tasks

Scheduled tasks allow you to set a maintenance window to start sometime in the future, so that proactive and pre-planned work can be carried out. Setting a future start time allows administrators to monitor servers up until any planned work is due to start, without having to do this manually.

Selecting this option, and clicking the Schedule Start/Update button will display a new window allowing you to enter the following details...

  • Task Name
    A title for the scheduled task,
  • Description
    A description for the task. The list of servers will be entered automatically,
  • Start Time
    When you want the scheduled task to start,
  • RunAs Username
    The username to use for this scheduled task. Make sure this user has logon rights to start scheduled jobs,
  • Password
    The password for the above user.
When you click Create, the application will:
  1. Save the currently entered server list into the same folder as the application, using a unique file name,
  2. Create a new Windows scheduled task with the name provided and the name of the file to be used.
    For example: SCOM Maintenance Mode Task (a54722x)
  3. Return you to the server list, so that you can create more scheduled tasks if required

Use Case Example

If there is planned work happening over the weekend, say a Sunday between 2am and 4am, you may want to put any affected servers into maintenance mode during this time

  • One way to do this is:
    1. Wake up at 1:30am,
    2. Logon to the corporate VPN,
    3. Remote into the SCOM Management server,
    4. Find the servers, one at a time,
    5. Put them into maintenance mode manually.
    At 1:30am, you could be half-asleep and mistakes might be made.
  • Another way could be:
    1. Pre-define the servers affected (during working hours),
    2. Wake up at 1:30am,
    3. Logon to the corporate VPN,
    4. Remote into the SCOM Management server,
    5. Put them into maintenance mode using this application.
    At 1:30am, you could be half-asleep and mistakes might be made.
  • With the scheduled task option:
    1. Pre-define the servers affected (during working hours),
    2. Run this application using the new scheduled task option to start at 1:30am (during working hours)
